# 💰 비트코인 트랜잭션 (Bitcoin Transaction) 

## 1. 핵심 암호화 기술 (Cryptography) 
- **DSA (Digital Signature Algorithm)** : 디지털 서명을 생성하고 검증하는 데 사용되는 공개 키 암호화 알고리즘이다. 거래 승인에 사용되는 개인키와 서명 검증에 사용되는 공개키 쌍을 생성한다.
- **디지털 서명 (Digital Signature)** : 특정 메시지가 개인 키 소유자에게서 나왔음을 인증하고, 메시지가 위변조되지 않았음을 확인하는 증명이다.
- **해시 함수 (Hash Function)**
  - SHA-256 : 서명 과정에서 데이터 무결성을 확인하는 데 사용된다.
  - SHA-1 : P2PKH 스크립트 OP_HASH160 연산을 시뮬레이션하기 위해 사용된다.

## 2. 비트코인 스크립트 기본 구조
비트코인 거래의 유효성을 검증하는 데 사용되는 스택 기반의 언어 구조이다. 
- **ScriptPubKey (잠금 스크립트)** : 코인을 인축하기 위해 충족해야 할 조건을 명시하며, 거래 출력에 포함된다.
- **ScriptSig (해제 스크립트)** : ScriptPubKey의 잠금 조건을 충족시키기 위한 증거를 제공하며, 거래 입력에 포함된다.

## 3. 거래 유형
1. P2PK (Pay-to-PubKey)
   
   공개키 자체를 잠금 조건으로 사용하는 가장 기본적인 거래 유형으로 코인을 사용하려면 해당 공개 키에 대한 유효한 서명을 제시해야 한다.
   - 잠금 조건 : "이 거래를 사용하려면, 이 공개 키로 생성된 유효한 서명을 제시해야 한다."
   - 특징 : 간단하지만, 공개 키가 트랜잭션의 ScriptPubKey에 직접 노출된다. 이는 비트코인 주소 (P2PKH) 방식보다 보안상 덜 선호된다.
   
2. P2PKH (Pay-to-PubKey-Hash)
   
   공개 키의 해시값, 즉 주소를 잠금 조건으로 사용하는 가장 일반적이고 표준화된 거래 유형이다. 잠금을 해제하려면 공개 키와 서명 모두를 제시해야 한다.
   - 잠금 조건 : "제시한 공개키가 잠금 장치에 설정된 해시값과 일치해야 하며, 그 공개키에 해당하는 유효한 서명을 제시해야 한다."
   - 특징 : 2단계의 검증을 거친다. 1) 주소 일치 확인과 2) 서명 유효성 확인으로 이는 P2PK보다 안전하며, 거래가 소비되기 전까지는 공개 키의 해시값만 노출되어 프라이버시가 더 보장된다.
